# 🚀 08 — Deployment Guide

Panduan deploy project ke production secara **100% gratis** menggunakan Vercel.

## 🎯 Quick Deploy ke Vercel

### Persiapan

1. **Akun Vercel** — sign up gratis di [vercel.com](https://vercel.com) (login dengan GitHub)
2. **Git Repo** — push project ke GitHub/GitLab/Bitbucket
3. **API Key** — pastikan punya Gemini API key

### Step 1: Push ke GitHub

```bash
# Init Git (jika belum)
git init
git add .
git commit -m "Initial commit: Plesir Pekalongan"

# Buat repo di GitHub, lalu:
git remote add origin https://github.com/username/plesir-pekalongan.git
git branch -M main
git push -u origin main
```

### Step 2: Import ke Vercel

1. Login ke [vercel.com/dashboard](https://vercel.com/dashboard)
2. Klik **"Add New Project"**
3. Pilih repo `plesir-pekalongan`
4. Framework Preset: Vite (auto-detected)
5. Build Settings (auto-detected):
   - Build Command: `npm run build`
   - Output Directory: `dist`
   - Install Command: `npm install`

### Step 3: Setup Environment Variables

Di halaman import, scroll ke **"Environment Variables"**:

```
Key: VITE_GEMINI_API_KEY
Value: AIzaSy...your_real_key_here
```

Klik **"Add"**, lalu **"Deploy"**.

### Step 4: Wait & Visit

Build memakan ~2-3 menit. Setelah selesai, dapat URL otomatis:
- `https://plesir-pekalongan.vercel.app`
- `https://plesir-pekalongan-username.vercel.app`

## 🌐 Custom Domain (Opsional)

### Opsi 1: Subdomain Vercel (Gratis Forever)

URL default sudah cukup untuk MVP/demo:
- `plesir-pekalongan.vercel.app`

### Opsi 2: Beli Domain `.id`

1. Beli di registrar seperti:
   - [pandi.id](https://pandi.id) — official `.id` registrar
   - [Niagahoster](https://niagahoster.co.id) — local
   - [Namecheap](https://namecheap.com) — international
   
2. Harga estimasi:
   - `.id` — ~Rp 250rb/tahun
   - `.co.id` — ~Rp 300rb/tahun
   - `.com` — ~Rp 150rb/tahun

3. Setup di Vercel:
   - Dashboard > Project > Settings > Domains
   - Add domain: `plesirpekalongan.id`
   - Vercel akan beri instruksi DNS records:
     ```
     Type: A
     Name: @
     Value: 76.76.21.21
     
     Type: CNAME  
     Name: www
     Value: cname.vercel-dns.com
     ```
   - Setup di panel domain provider
   - Tunggu propagasi DNS (5 menit - 24 jam)
   - SSL auto-generated by Vercel

## 🔧 Build Optimization

### Vite Config untuk Production

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import { VitePWA } from "vite-plugin-pwa";
import { visualizer } from "rollup-plugin-visualizer";

export default defineConfig({
  plugins: [
    react(),
    VitePWA({ /* config dari docs/06 */ }),
    visualizer({ open: false, filename: "dist/stats.html" }),
  ],
  build: {
    target: "es2020",
    minify: "terser",
    terserOptions: {
      compress: {
        drop_console: true, // Hilangkan console.log di production
        drop_debugger: true,
      },
    },
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ["react", "react-dom", "react-router-dom"],
          ui: ["framer-motion", "lucide-react"],
          maps: ["leaflet", "react-leaflet"],
          gemini: ["@google/generative-ai"],
        },
      },
    },
    chunkSizeWarningLimit: 1000,
  },
  server: {
    port: 3000,
    open: true,
  },
});
```

### Bundle Analysis

Check bundle size:

```bash
npm run build
# Buka dist/stats.html untuk visualisasi
```

Target:
- Initial JS bundle: < 200 KB gzipped
- LCP: < 2.5s
- TTI: < 3.8s

## 🔍 SEO Setup

### 1. Sitemap.xml

Generate otomatis dengan `vite-plugin-sitemap`:

```bash
npm install -D vite-plugin-sitemap
```

```typescript
// vite.config.ts
import Sitemap from "vite-plugin-sitemap";

plugins: [
  // ...
  Sitemap({
    hostname: "https://plesirpekalongan.vercel.app",
    dynamicRoutes: [
      "/destinasi",
      "/event",
      "/berita",
      "/rencana",
      "/kontak",
      // ... dynamic routes dari data
    ],
  }),
]
```

### 2. robots.txt

```
# public/robots.txt
User-agent: *
Allow: /

Sitemap: https://plesirpekalongan.vercel.app/sitemap.xml
```

### 3. Meta Tags per Halaman

```tsx
// Setiap halaman pakai react-helmet-async
import { Helmet } from "react-helmet-async";

export function Home() {
  return (
    <>
      <Helmet>
        <title>Plesir Pekalongan — Jelajahi Pesona Kota Batik Dunia</title>
        <meta name="description" content="Website pariwisata resmi Kota Pekalongan. Jelajahi 50+ destinasi wisata, event budaya, dan kuliner khas." />
        <meta property="og:title" content="Plesir Pekalongan" />
        <meta property="og:description" content="Jelajahi Pesona Kota Batik Dunia" />
        <meta property="og:image" content="/og-image.jpg" />
        <meta property="og:url" content="https://plesirpekalongan.vercel.app" />
        <meta name="twitter:card" content="summary_large_image" />
      </Helmet>
      {/* ... */}
    </>
  );
}
```

### 4. Schema.org JSON-LD

Untuk destinasi & event:

```tsx
<Helmet>
  <script type="application/ld+json">
    {JSON.stringify({
      "@context": "https://schema.org",
      "@type": "TouristAttraction",
      "name": destinasi.nama,
      "description": destinasi.deskripsi_singkat,
      "image": destinasi.foto_cover,
      "address": {
        "@type": "PostalAddress",
        "addressLocality": "Pekalongan",
        "addressRegion": "Jawa Tengah",
        "addressCountry": "ID",
      },
      "geo": {
        "@type": "GeoCoordinates",
        "latitude": destinasi.lokasi.koordinat.lat,
        "longitude": destinasi.lokasi.koordinat.lng,
      },
    })}
  </script>
</Helmet>
```

### 5. Submit ke Search Console

1. Buka [search.google.com/search-console](https://search.google.com/search-console)
2. Add property: `https://plesirpekalongan.vercel.app`
3. Verify via DNS TXT record atau HTML tag
4. Submit sitemap: `https://plesirpekalongan.vercel.app/sitemap.xml`

## 🛡️ Security Headers

Buat file `vercel.json` di root:

```json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "X-Content-Type-Options",
          "value": "nosniff"
        },
        {
          "key": "X-Frame-Options",
          "value": "DENY"
        },
        {
          "key": "X-XSS-Protection",
          "value": "1; mode=block"
        },
        {
          "key": "Referrer-Policy",
          "value": "strict-origin-when-cross-origin"
        },
        {
          "key": "Permissions-Policy",
          "value": "camera=(), microphone=(), geolocation=(self)"
        }
      ]
    }
  ]
}
```

## 🆘 Troubleshooting

### Build Fails di Vercel

**Symptom:** "Module not found" atau type errors

**Fix:**
```bash
# Test build locally dulu
npm run build

# Kalau ada error TS, fix dulu
npm run type-check
```

### API Key Tidak Bekerja di Production

**Symptom:** Gemini error 401 / 403

**Fix:**
1. Check Vercel Dashboard > Settings > Environment Variables
2. Pastikan variable name: `VITE_GEMINI_API_KEY` (prefix `VITE_` wajib!)
3. Redeploy setelah update env var

### Images Tidak Load

**Symptom:** Unsplash images 403 / broken

**Fix:**
- Unsplash source.unsplash.com terkadang slow/rate-limited
- Cache images saat dev: copy ke `/public/images/`
- Atau gunakan CDN gambar lain

### Route Refresh 404

**Symptom:** Refresh di `/destinasi` returns 404

**Fix:** Buat `vercel.json`:
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```
